autoRfam help


What is autoRfam?

It's a collection of automatically built alignments from RNAcentral sequences, which could pontentially be used to build new Rfam families.


Fields in the browse alignments table

Alignment
name of stockholm alignment file
The alignment inherits the name of the RNAcentral sequence to which the rest of the sequences were aligned.
It corresponds to the first sequence of the alignment, and it's given automatically by nhmmer.

Selected description
name of the first sequence of the alignment
This description comes from RNAcentral, and it is taken out of the FASTA file.

Number of sequences
in the alignment

Alignment length
which considers sequence and gaps

Avg. sequence length
the average length of the sequences with no gaps

Lengths ratio
Avg. sequence length / Alignment length
A way of representing how gapped an alignment is.

Number of publications
distinct number of publications related to the sequences in the alignment
Extracted from RNAcentral for each sequence, only those with pubmed_id are included.

Number of databases
distinct number of databases referencing the sequences in RNAcentral

RNAcode warning
indicates if significant coding regions were found for the alignment
Using the RNAcode software

R-scape output
indicates if statistically significant basepairs were found for the alignment
Using the R-scape software

How were the alignments made?

We selected all the sequences from RNAcentral that couldn't be matched to any existing Rfam family. Then, with nhmmer we produced an alignment per sequence including only significant hits.
To have a non-redundant set of alignments, we clustered the ones that shared sequences and selected only one alignment per cluster. The criteria was to take the 3 alignments with more sequences in them, and then take the one with the largest "Lengths ratio".


What's with the .cl extension?

The .cl extension is added into an alignment if it was processed by a cleaning up script. When nhmmer aligns the sequences, it might include several subsections of the same URS in a single alignment. The cleaning up script atempts to remove these repeated sequences.The name is edited to (1) indicate that it differs from the original output alignment and (2) because, in some cases, it might be an indication of strange behaviour such as repeats.
